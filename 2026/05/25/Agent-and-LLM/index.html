1. 什么是Function Calling

过去 ：LLM需要调用工具，输出的自然语言，harness通过手写多个if-else来匹配 or 猜测LLM想调用的工具。

问题 ：LLM每次输出的文本不一定，可能出现格式错误、参数错误、甚至拼写错误、大小写错误等问题，与写死的if-else不一定能完全匹配，可靠性低。

解决方案： Function Calling。由harness主动提供所有可调用的工具，让LLM自主决策选择工具调用。其中每个工具定义都是统一的Json Schema格式，LLM调用工具时输出结构化的Json调用请求。

为什么更可靠：Function Calling 不是靠”猜”文本。工具定义通过 API 传给模型，模型在训练时就被要求输出合法 JSON，API 层还会校验 schema。输出格式由模型保证，不再依赖 harne ss 的正则匹配。

response.content 中的每个 block 有两种类型：

ToolUseBlock（工具调用）

并行调度：一次回复可调度多个工具，减少交互轮次。多个独立操作一次发起，互不等待。

2. Agent 和 LLM 的区别

LLM是推理引擎，Agent是自动化系统。

LLM是本质是一个单次对话的语言生成模型，只能做单次推理，但是没有记忆、没有执行能力、不知道怎么调用工具、无法感知环境。

Agent将LLM作为大脑，在外层包装了一个循环，它维护对话历史，提供记忆功能、它能够注册并调用工具、它能够管理上下文、它能感知环境实现闭环，不断循环直到任务彻底完成。

3. LLM如何学会调用外部工具

第一个阶段：SFT

有监督微调，每条样本包括system prompt、工具定义、用户提问、模型应该输出的JSON结构化调用格式、工具执行结果、模型最终输出结果。

训练样本应该覆盖：单工具调用、多工具并行调用、工具调用后失败处理情况、不需要调用工具直接回答、多轮对话中的工具调用。

SFT使模型学会如何调用工具，但样本中正确调用的例子偏多，模型可能偏激，总是想调用工具。

第二个阶段：RLHF

用奖励模型的打分不断优化主模型参数，使模型建立边界，学会什么时候该调工具，什么时候直接回答。
